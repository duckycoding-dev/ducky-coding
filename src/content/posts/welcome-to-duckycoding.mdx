## The Tech Stack That Powers This Blog

### üöÄ Astro - The Foundation

The heart of this blog is built with [Astro](https://astro.build), and I couldn't be happier with this choice. Here's why Astro was perfect for this project:

- **Content-First Architecture**: Astro is specifically designed for content-heavy sites like blogs
- **Island Architecture**: Components are hydrated only when needed, resulting in blazing-fast performance
- **Framework Agnostic**: I can use React, Vue, or any other framework when needed, but most of the site is pure HTML/CSS
- **Excellent Developer Experience**: Hot reloading, TypeScript support out of the box, and intuitive file-based routing

### üìö Content Collections - Structured Content Management

One of Astro's killer features is Content Collections, which I'm using to manage blog posts and topics:

```typescript
const posts = defineCollection({
  loader: glob({ pattern: '**/*.{md,mdx}', base: './src/content/posts' }),
  schema: PostContentSchema,
});

const topics = defineCollection({
  loader: glob({ pattern: '**/*.json', base: './src/content/topics' }),
  schema: TopicContentSchema,
});
```

This gives me:

- **Type Safety**: Every post is validated against a Zod schema
- **Automatic Routing**: Posts are automatically available at `/posts/[slug]`
- **Rich Metadata**: Frontmatter includes everything from reading time to featured status
- **MDX Support**: Full MDX capabilities when I need interactive components in posts

### üóÑÔ∏è SQLite + Drizzle ORM - Simple Yet Powerful Data Layer

For the database layer, I went with SQLite managed through Drizzle ORM:

**Why SQLite?**

- Perfect for a blog that doesn't need complex relationships
- Zero configuration - it's just a file
- Incredibly fast for read-heavy workloads
- Easy to backup and migrate

**Why Drizzle ORM?**

- TypeScript-first approach with excellent type inference
- Lightweight and performant
- SQL-like syntax that doesn't hide the database
- Great migration system

```typescript
// Example of how clean the database operations are:
const posts = await db
  .select()
  .from(postsTable)
  .where(eq(postsTable.status, 'published'))
  .orderBy(desc(postsTable.createdAt));
```

### Content Management Workflow

Writing a new blog post is as simple as:

1. Create a new `.mdx` file in `src/content/posts/`
2. Add the required frontmatter (validated by Zod schema)
3. Write the content in Markdown/MDX
4. The post automatically appears on the blog with proper routing, SEO, and styling

### Performance Optimizations

- **Static Site Generation**: Most pages are pre-rendered at build time
- **Image Optimization**: Automatic image optimization with Astro's built-in tools
- **CSS Purging**: Unused CSS is automatically removed in production
- **Minimal JavaScript**: Only interactive components are hydrated
